Difference between compiler and interpreter:
S.N. Compiler Interpreter
1 It translates the whole program into object code at a time. It translates one line or a single statement of a program into object code at a time.
2 It finds out the error after compiling the complete program. It finds out the error after translating a line of the program at a time.
3 The translating process is incredibly faster. The translating process is slower.
4 Compiler based program is difficult to code and debug. Interpreter based program is easy to code and debug.
5 It is usually a large program and requires larger memory space. It is usually a smaller program and requires less memory space.
6 New programming languages don’t use compiler. New programming languages use interpreter.
7 For e.g.: C, C++, Java, FORTRAN etc. For e.g.: BASIC, C#, PHP etc.
Syntax and Semantics:

As we know, programming languages help the user communicate with the computer. The user is governed by several sets of codes that instruct the computer to perform specific tasks. This set of codes and its structure is known as syntax. In addition, semantics refers to the meaning attached to individual words or symbols in a programming language. For example:

In QBasic:

Print “This is programming”; P

Here, "Print" is semantics, and the entire line is the structure (syntax).

Error (BUG):

There may be several errors inside a program that stop it from being executed. An error in a program is known as a bug. This type of error may arise from not following proper rules given by the programming language, poor problem analysis, or hardware failure. Errors in a program can give ambiguous (not accepted/false) results. The process of removing errors from a computer program is known as debugging. There are three types of errors:

  • Syntax error: The error arises due to not following the proper rules or format given by the programming language. Each programming language has its own vocabulary, punctuation, and structure. This error can be detected by language processors such as compilers and interpreters. For example, in C, every statement should end with a semicolon; otherwise, a syntax error occurs.
  • Logical error: This occurs when the problem is poorly understood, hampering the solution. This error is not detected by language processors. For example, in calculating simple interest, if the formula used is I = P * T * R, the logical error will occur.
  • Runtime error: These errors occur when the program is running. They are not detected by the language processor. Examples include insufficient memory space or a peripheral device not functioning.
Some common terminology in programming:

Data types: Data types specify the type of data to be used in a program. Common data types are:

  • Numeric: Positive and negative whole numbers, including zero.
  • Character: Alphanumeric values including special symbols.

Variable and Constant:

  • Variable: An entity that holds numeric or alphanumeric values and changes its value during the program execution.
  • Constant: An entity that holds numeric or alphanumeric values but does not change its value during the program execution.

Operands and Operators and Operations:

Let us consider: S=A+B

  • Operands: S, A, B
  • Operators: +, =
  • Operation: The sum of two operands, A and B, which is an operation.
Types of operators:
  • Arithmetic operators: +, -, /, *
  • Relational operators: <, >, >=, <=, =
  • Logical operators: AND, OR, NOT
Program Design Tool:

There are several program design tools available. Some commonly used program design tools are algorithm, flowchart, and pseudocode.

Algorithm:

The step-by-step procedure or sequence of steps for solving a specific problem is known as an algorithm. Algorithms are written in simple English language and are not closely related to any programming language.

Rules for writing an algorithm:
  • Algorithm should produce output after a finite number of steps.
  • Algorithm should not depend on any specific programming language.
  • Algorithm should terminate after a finite number of steps.

    If yes, Display the greatest number is “a” and go to step 4

What is C programming language?

C is a High-Level Programming Language developed by Dennis Ritchie at AT&T's Bell Laboratories in 1970. It is widely used for both system and application programming. Known as a Middle-Level Programming Language, it works with both low-level and high-level language features.

What are the features of C language?

  • Supports both high-level and low-level programming, making it a middle-level language.
  • Primarily used for system programming but also widely used for application programming.
  • Efficient and fast program development.
  • Includes powerful tools and keywords.
  • Has its own text editor and compiler.
  • Supports structured programming.
  • Highly portable.
  • Comes with a vast set of library functions.
  • Flexible for programming.

Advantages and Disadvantages of C language:

Advantages Disadvantages
Suitable for system programming. No runtime checking.
Easier interaction with hardware. No strict data type checking.
Easy to learn. Debugging can become complex with large programs.
Compact and efficient code. Requires semicolons at the end of each statement.

Define the following terms:

  • Editor: The area where the program is typed. C has its own text editor, but other text editors can also be used.
  • Pre-processor: A collection of instructions, written with a preceding # symbol, to help the machine understand certain keywords (e.g., #include).
  • Compiler: A software that converts high-level code into machine-readable object code.
  • Linker: A software that links various library files with object codes to create an executable file (.exe).

Character Set in C Language:

  • Alphabets: A-Z and a-z.
  • Digits: 0-9.
  • Special Symbols: ~ ' ! @ % ^ & * ( ) - _ + = | \ { } [] : ; " ' <> , . ? /

Comments in C:

  • Single-line comment: Written with the // symbol.
  • Multi-line comment: Written with /*...*/ symbols.

C Tokens: A set of identifiers, keywords, constants, variables, and operators that form the smallest unit in C programming. These units are not broken down during compilation.

Data Types in C:

  • int: Represents integer data. Uses 2 bytes of memory.
  • float: Represents real numbers with decimal points. Uses 4 bytes of memory.
  • char: Represents a single character. Uses 1 byte of memory.
  • void: Represents no value or null value.

Escape Sequences in C:

Character Escape

Array

An array is a collection of variables of the same type represented by a common name. It contains homogeneous data (similar data types). If we declare an array as int, it must only contain integer data, and the same applies to floats and char. Each array element is accessed using the array name followed by an index enclosed in square brackets (e.g., a[10]). Hence, it is also called an indexed or subscripted variable.

Example: int age[5];

This means the array name is age, capable of storing five different integer values.

Types of Arrays

  • 1-Dimensional
  • N-Dimensional (Multi-Dimensional)

Q5: WAP to sort 10 numbers in ascending order.

#include <stdio.h>
int main() {
    int a[10], i, j, temp;
    printf("Enter 10 numbers: ");
    for(i=0; i<10; i++) {
        scanf("%d", &a[i]);
    }
    for(i=0; i<9; i++) {
        for(j=0; j<9-i; j++) {
            if(a[j] > a[j+1]) {
                temp = a[j];
                a[j] = a[j+1];
                a[j+1] = temp;
            }
        }
    }
    printf("Sorted numbers: ");
    for(i=0; i<10; i++) {
        printf("%d ", a[i]);
    }
    return 0;
}

These arrays have more than one subscript.

#include <stdio.h>

        for(j=0; j<3; j++) {
            p[i][j] = 0;
            for(k=0; k<3; k++) {
                p[i][j] += a[i][k] * b[k][j];
            }
        }
    }
    printf("Multiplied Matrix:\n");
    for(i=0; i<3; i++) {

            printf("%d\t", p[i][j]);
        }

    }
