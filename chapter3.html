Chapter 3:Computer Software and Operating System

Introduction

As we know, stand-alone hardware cannot do anything. For the smooth operation of our computer system, there should be proper coordination between hardware and software. Software helps to mobilize the hardware and other resources. In order to mobilize hardware, we have to write several sets of instructions that instruct the computer what to do and what not to do. These sets of instructions are collectively known as a program, and the term software is the collection of related programs and associated documents. In order to produce useful output, hardware and software must work together. Nothing useful can be done with hardware only, and software cannot be utilized without hardware. The different types of software are:

  1. System software: System software is a set of one or more programs designed to control the operation of our computer system. This type of software doesn't fulfill the specific requirements of the user. They are general programs written to assist humans in the use of the computer system. In general, system software supports the running of other software, communicates with peripheral devices, supports the development of other types of software, and monitors the use of various hardware resources. Thus, the system software makes the operation of the computer system more effective and efficient. The system software is categorized into two categories:
    1. System Management: It is responsible for the proper management and functioning of the computer system. All types of management between computer hardware and software are performed by system management software. There are 3 types of system management software:
      1. Operating System: An operating system is a collection of programs that controls the overall operation of the computer system. It is the first program that is loaded into memory when the computer is turned on. It provides a platform for other application programs to run and execute. It provides the user with an interface so that users can easily communicate with the computer. For example, Linux, Windows, Mac, etc.
      2. Utility software: It is the supporting software that is used to perform specific tasks related to the maintenance of the computer system. Some of the utility software is included in the operating system, whereas some are available as separate utilities in the market. They are also called service programs. For example, Norton Utility, PC Tools, WinZip, etc.
      3. Device driver: A device driver is software that is responsible for the smooth functioning of the hardware device connected to the computer. When we add a new device to the computer, we need to install new software called a device driver. The device driver will coordinate with the operating system so that the newly installed hardware functions properly and smoothly.
    2. System Development: It refers to a set of programs that are used to develop computer programs. This type of software is not used by a normal user of the computer. They are used by programmers for the development of new programs. There are two types of system development software:
      1. Programming language: The language with which we give instructions to the computer is known as a programming language. Programming languages are sets of different keywords, variables, operators, loops, and other symbols. They help to make communication between the computer and the user. There are two types of programming languages: low-level language and high-level language.
      2. Language translator: It is a special kind of computer software that translates programs written in one language into another language. It is compulsory for both low-level and high-level languages. It is also called a language processor. The types of language translators are compilers, interpreters, and assemblers.
  2. Application Software: Application software is a set of one or more programs designed to do a specific task. It is made to fulfill user demands. This program directs a computer to solve user-oriented problems such as preparing bills, calculating mathematical equations, preparing mark sheets, etc. The software that is developed for user purposes is called application software. These types of software are generally developed using high-level languages. For example, MS Excel, Photoshop, billing software, etc. There are two types of application software:
    1. Packaged software: They are generalized sets of programs designed and developed for general purposes. It is generally large-sized, error-free, advanced, and standard software with much more functionality for special work. This type of software doesn't perfectly match the requirements of many organizations or users. They cannot be changed easily and are developed and designed by reputed software companies, so they are trustworthy. They are expensive for small organizations. For example, MS Office, Adobe, Macromedia, etc.
    2. Customized/Tailored software: This is software developed in a high-level language for special tasks. This type of software is developed for specific purposes to solve specific problems of specific users or organizations. The requirements of the user or organization can be perfectly matched. They can be changed easily since they are made by local programmers. For example, payroll systems, inventory management, school management, billing software, mark sheet evaluation, etc.

Operating System - Introduction: An operating system (OS) is an integrated set of programs that controls overall resources such as CPU, memory, and input-output devices of the computer system. The major objective of an operating system is to improve the performance and efficiency of a computer system. Like the manager of a company, an operating system is responsible for the smooth and efficient functioning of the entire computer system. The operating system provides the platform for other application programs/software to run and execute. It provides the user with an interface so that the user can easily communicate with the computer, which is more convenient to use and operate.

An operating system is a collection of programs that controls the overall operation of the computer system. It also controls and coordinates the use of hardware among various application programs for various users. So, it acts as an interface between the user and the computer hardware. The other major functions of an operating system are:

  • It controls, monitors, and coordinates the overall operation of our computer system.
  • It acts as an interface between the user and the computer.
  • It provides a platform to develop, run, and execute other computer programs.
  • It manages hardware resources such as CPU, memory, input-output terminals, networking equipment, etc.
  • It hides programming and hardware complexity from the user.

The primary goal of an operating system is to maximize the productivity of a computer system by operating it in the most efficient manner and maintaining the amount of human intervention. Some examples of operating systems are: MS-DOS, Windows, Linux, macOS, UNIX, Chrome, Fedora, Ubuntu, etc.

Functions / Features / Advantages of Operating System (OS)

  1. Input-Output (I/O) Management: Input-Output is essential to operate any computer. It allows the computer to interact with several peripheral devices such as keyboards, mice, printers, scanners, etc.
  2. User Interface (UI): User Interface means an ideal environment in which the user can work so that they can interact with the computerized system. Every operating system provides the feature of a user interface to enhance the experience and joy of using a computer or any other computerized system. It acts as a bridge between the user and the computer. There are 2 types of user interfaces:
    • Character/Command User Interface (CUI): e.g., MS-DOS
    • Graphical User Interface (GUI): e.g., Windows, macOS
  3. Security: The operating system of a computer has several built-in tools to protect against security threats such as viruses, unauthorized access, suspicious network activity, etc. The basic security in an OS is to control access to your computer by setting users and passwords.
  4. Process Management: The process management allocates a processor to execute a chosen process. The OS acts as a traffic controller, job scheduler, process scheduler, and dispatcher.
  5. Memory Management: Memory is a large array of bytes, each with its own address. When the user requests the CPU for read-write operations, the OS determines the amount of memory required for the program instructions and data. Then, the OS allocates the required memory to load the program and data into RAM. When the program terminates, it frees up the space, and a new program is loaded.
  6. Data Management: Data management allows organizing data into logical groupings called files. Earlier operating systems did not provide features of data management. Hence, they were inflexible, but nowadays, every operating system provides this feature.
  7. Command Interpreter: The command interpreter reads the commands that a user types in at a terminal, interprets them, and translates them into a detailed set of instructions that computer hardware can understand. Every operating system must have a command interpreter for its operation.
  8. Deadlock Prevention: During processing, a situation can arise in which a resource shared by two or more processes cannot continue because the resource required by one process is held by another. This situation is known as a deadlock. For example, if Process-1 allocates Resource A and later requires Resource B, and Process-2 allocates Resource B and later requires Resource A, neither Process-1 nor Process-2 will be executed. Such a situation is called a deadlock. The operating system ensures the prevention of deadlock by taking situational action through careful allocation of resources.
  9. Time Sharing: The function of an OS that involves the CPU allocating time to a number of users on the same computer. This property is generally found in network operating systems such as Windows NT.
  10. Virtual Memory: Virtual memory is a feature of an operating system that allows a computer to compensate for a shortage of physical memory by transferring temporary files from RAM to disk. Virtual memory has twice as many addresses as main memory. The process of translating virtual addresses into real addresses is known as mapping. The copying of virtual pages from disk to main memory is called swapping.

Types of Operating System (OS)

Operating systems can be classified based on the following criteria:

  • Basis of Processing
  • Basis of User Interface (UI)
  • Basis of Mode of User

A) On the Basis of Processing

  1. Multi-programming OS: Multi-programming is used in a multi-user environment. It is the technique in which multiple user programs are executed simultaneously by a single processor. Multi-programming means that when two or more programs are provided to the CPU for processing or loaded into the internal storage of the CPU at the same instant of time, the portion of one program is executed, then the portion of another program is executed, and so on. Thus, multi-programming refers to the concurrent execution of several programs. The main purpose of multi-programming is to increase the utilization of the computer and its resources. Different forms of multi-programming OS are multi-tasking, multi-processing, and multi-user.
  2. Multi-tasking OS: A multi-tasking OS allows more than one program to run concurrently, mainly in a single-user system. These computers are capable of executing several tasks or programs at the same time. For executing several tasks, it can be a single or multi-processing system. In some multi-tasking systems, one of the processes is called the foreground (active), and others are called the background (inactive). Multi-tasking OS increases the productivity of the user by executing many tasks simultaneously. Some of the most common examples are MS Windows, Linux, macOS, etc.
  3. Multi-processing OS: A multi-processing system has more than one processor linked in a coordinated way. A multi-processing OS supports running a program on more than one CPU. Generally, super servers are especially designed to support multiple processors. UNIX is a commonly used multi-processing OS. Multi-programming and multi-processing are not the same concepts. Multi-programming involves the concurrent execution of two or more programs by a single CPU, whereas multi-processing involves two or more processors (CPUs) for controlling different activities or the execution of many programs simultaneously. The multi-processing concept is also known as parallel processing. It is a more costly and complex technique than multi-programming, which dramatically increases the program execution speed of the computer.
  4. Time-Sharing OS: Time-sharing is a technique that enables many people located at various terminals to use a particular computer system at the same time. Processor time, which is shared among multiple users simultaneously, is termed time-sharing. Multiple jobs are executed by the CPU by switching between them, but the switches occur so frequently that the user can receive an immediate response.
    • Advantages: Provides quick response and reduces CPU idle time.
    • Disadvantages: Problems of reliability, less security, and integrity.
  5. Batch Processing OS: Batch processing is a technique that requires grouping similar jobs, which consist of programs, data, and system commands. It is also known as offline processing. This type of processing is suitable for programs with large completion or involvement. For example, payroll, weather forecasting, statistical analysis, and many more. Users need not wait while the job is being processed. They can submit their program to the operator and collect it later, meaning that the jobs are entered, stored on a disk in a batch or queue, and executed one after another under the control of the OS.

    The major disadvantage of batch processing is that it does not allow interaction between the user and the program during execution.

  6. Multi-threading OS: A program in execution is known as a process and can be divided into multiple smaller sub-processes. These sub-processes are known as threads. A multi-threading OS has the ability to divide processes into threads and execute them concurrently. Threads are individual processes that execute simultaneously in a multi-tasking OS.
  7. Real-time OS: Real-time processing is a method that controls the environment by receiving data and taking action quickly to affect the functioning of the environment at the same instant of time. It is an online processing system in which processing time is critical. The entire processing task has to be completed within the predefined time. The real-time OS is suitable for the following types of operations:
    • Airlines or flight seat availability
    • Rocket launching systems
    • Wholesale suppliers and manufacturers, etc.
  8. Online Processing OS: It is a popular processing technique at present. In this method, transactions are processed as soon as they happen at the place of their origin. It is quite simple compared to a real-time OS. In this type, the user can interact or provide input during processing as well. The output is provided back to the user as soon as the processing is completed.

B) On the Basis of User Interface (UI)

A user interface allows the user to interact with the computer system. It provides the user with an ideal environment so that the user can experience better computing. It brings structure to the interaction between the user and the computer. Without a user interface, it is almost impossible to operate a computer smoothly. Hence, in order to enhance the experience of using a computer, the UI plays a vital role. Mainly, there are 2 types of user interfaces (UI):

  1. Character User Interface (CUI): It is a type of operating system in which the whole computer operates or depends upon the commands issued by the user. All the operations that take place in a CUI have their own specific commands. Hence, they are known as Character User Interfaces. The user has to remember all the commands needed to operate the computer. So, they are non-user-friendly. Only people with technical knowledge can operate this type of operating system. It is a primitive type of OS in which graphics, pictures, icons, drop-down menus, dialogue boxes, etc., are not used, making it more difficult to use. It uses the character or text mode of the computer, such as DOS and UNIX. A command prompt is a sequence of characters used in a CUI. Commands are typed in to instruct the computer. MS-DOS is a Character User Interface operating system.

    The major advantage of this type of OS is that it can have a quick response as long as the user enters the correct command. However, the disadvantage of a command-driven user interface is that it is very difficult to use if the user is a beginner and doesn't know the correct command. However, it can be used in older systems with no processing capabilities and low storage.

    Features of CUI:

    • It is a single-user and single-tasking operating system.
    • Less user-friendly than GUI.
    • Commands are used to instruct the computer.
    • The user needs to remember the commands for operating this system.
    • It cannot display graphics, icons, pictures, or multimedia.
    • It is faster than a GUI system.
    • Pointing devices such as mice, joysticks, and trackpads are not recognized.
    • It can be used in low-memory and low-processing computers.

  2. 2) Graphical User Interface (GUI):

    It is a type of operating system in which computers are operated just by clicking or selecting the required icon or dialogue box instead of writing a direct command to the system. Since graphics like icons, pictures, drop-down menus, dialogue boxes, etc., are used, it makes it more user-friendly than CUI. Users don't have to remember a single command, so even users without technical knowledge can use this type of OS. Due to its user-friendly interface, it is simple to use and widely used for domestic applications. The goal of a graphical operating system is to create a system that the computer user can turn on and operate in the right way without any training or by remembering commands. The main advantage of GUI is that it is easy to use, especially for beginners. The main disadvantage of GUI is that it consumes more memory for operation. Examples of GUI include Windows and macOS.

    The first Graphical User Interface was designed by Xerox Corporation's Palo Alto Research Center in the 1970s, but it was not popular until the 1980s when Apple Macintosh, which used GUI, became popular.

    Features of GUI:

    • It is user-friendly, so it is easy to use.
    • Users don't have to remember commands.
    • It supports full multimedia and animation.
    • It requires larger memory for its processing.
    • GUI environments support full multimedia and animation.
    • It can be used in a network environment.
    • It is a multi-tasking, multi-programming, multi-user operating system.
    • Every type of peripheral device can be used.

    GUI Environment Components:

    • Icon: Icons are small pictures that represent commands, files, or windows.
    • Pointer: A symbol (usually an angled arrow) that appears on the display screen and is moved to select objects and commands. It is controlled by a pointing device such as a mouse, trackpad, or light pen.
    • Desktop: The area on the display screen where icons are grouped is referred to as the desktop.
    • Windows: Windows are displayed on the screen, and their shape and size can be changed at will. Each window can run a different program or display different information.
    • Menus: Most graphical user interfaces allow users to execute commands by selecting a choice from a menu.
    • Folders: Graphical representation of a directory.

    Difference Between Character User Interface and Graphical User Interface [CUI vs GUI]

    Character User Interface [CUI] Graphical User Interface [GUI]
    In CUI, the user has to interact with applications by using commands. In GUI, the user interacts with applications by using graphics.
    CUI stands for Character User Interface. GUI stands for Graphical User Interface.
    In CUI, only one task can run at a time (single-tasking). In GUI, more than one task can run at a time simultaneously (multi-tasking).
    It is not a user-friendly interface. It is a very user-friendly interface.
    The user does everything by using commands. Everything is interacted with by pointing at the application using devices like a mouse.
    Faster than GUI. Slower than CUI.
    Older computers were operated using CUI. Modern computers use GUI.
    Examples: DOS, UNIX, etc. Examples: Windows, macOS, etc.

    C) On the Basis of Mode of User

    There are two types of operating systems based on the mode of user:

    1. Single User: A single-user OS is a type of operating system that is developed and intended for use on a computer or similar machine that will only have a single user at any given time. This is the most common type of OS used on home computers as well as on computers in offices or other work environments. There are two general types of single-user systems: single-user single-tasking and single-user multi-tasking. Examples of single-user single-tasking operating systems are not much feasible for general-purpose computing. For example, MS-DOS, PC-DOS, etc.
    2. Multi-User: A multi-user OS is a computer operating system that allows multiple users on different terminals to access a single system with one OS. A multi-user OS differs from a single-user system on a network in that each user is accessing the same OS on different machines. The multi-user OS shares computer resources among different users, allowing each user a small slice of the processor time. This concept is also known as time-sharing. Due to this, it gives the user the impression that they are all being served simultaneously, even though the OS serves them one at a time. Examples include Linux and UNIX.

    Some Important Operating Systems in Use

    Open Source Operating System: Open source generally refers to applications whose source code is freely available on the internet to worldwide users. Basically, software that falls under the General Public License is open-source software. Users can customize the source code of open-source software according to their requirements. They need not buy a license to use the software. The source code can be modified and redistributed to any other person legally. This software is free of cost. Some examples of open-source software are Linux, Unix, Mozilla, Apache, etc. Similarly, open-source operating systems are those operating systems whose program code is open to all users. Some common examples of open-source operating systems are Linux, Unix, Minix, OpenSolaris, etc.

    UNIX: UNIX is a powerful, flexible, and manageable operating system with good utilities and communication abilities. It is a multi-user, multi-tasking, and multi-processing operating system. It is used in bigger machines like mainframes and minicomputers. It was developed by AT&T Bell Labs around 1970. UNIX is developed in C and Assembly language. The development of UNIX has given birth to many operating systems such as Linux and Solaris. These operating systems inherit their main features from UNIX.

    Linux: It is an open-source, powerful UNIX-based operating system that can run on a variety of platforms, including Intel, Power PC, DEC Alpha processors, as well as multi-processing systems. It was developed by Linus Torvalds at the University of Helsinki as a college project. He released the version of Linux, including all the source code. It is a user-developed product, meaning that users around the world who run this OS for their own use have developed many of its components and drivers. It is a multi-user, multi-tasking, and multi-programming OS, mainly popular for server systems. It is distributed through different distributors such as Red Hat, OpenSUSE, Ubuntu, Fedora, Debian, Granular Linux, Mandriva, etc.

    macOS: macOS is a popular GUI-based OS for Apple Inc. It is used in Apple Macintosh computers. It has a very high-quality graphical user interface. Earlier versions of this OS were only compatible with Motorola 68000 series processors, but now they are also compatible with Intel processors.

    Some Terminologies Related to Operating Systems

    Spooling: This process is used to balance the difference in speed between the high-speed CPU and slow-speed input-output devices. It is primarily used for printing purposes. When we provide multiple files to print to a printer together, although the printer can print one file at a time, all files get lined up in a queue and are printed one by one. In this process, the CPU remains free, and the user can work on other applications. This technique of printing is possible through a capability of the operating system known as spooling. The term spooling is derived from the term SPOOL (Simultaneous Peripheral Operation On-Line).

    Process: A process is basically a program in execution. It doesn't mean a program. A program itself is static, while a process is dynamic. For simplicity, we can define a program as a job that resides in secondary storage (not loaded into main memory). A process is a program that is currently in a running or ready state (loaded into main memory). A process in a computer may be in different possible states:

    • Running State: A process is said to be in a running state if it is currently running on the processor.
    • Ready State: A process is said to be in a ready state if it is temporarily stopped to let other processes run on the processor.
    • Blocked State: A process is said to be in a blocked state if it is unable to run until some external event occurs.

    Thread: A single process is divided into multiple sub-processes, and even if one part of the process goes for input-output operations, another part of the same process starts executing. This minimizes the burden of context switching to some extent. When multiple processes are loaded into memory at once, and one process goes into a blocked or ready state, another process is chosen by the scheduler to be in the running state. During this phase, the current state of the previously running process must be stored somewhere so that it can resume from that state the next time. This mechanism is known as context switching. It is a very costly operation and reduces efficiency. To solve this problem, newer operating systems use the mechanism of threading (lightweight processes).

    Scheduling: When more than one process is running, the operating system must decide which one to run first. The part of the OS concerned with this decision is called the scheduler, and the algorithm it uses is called the scheduling algorithm. Scheduling can be of two types:

    • Priority Scheduling: Each program or job is assigned a priority set by the program. In this scheduling, the job or program with the highest priority is treated first or selected to run. The only disadvantage of this scheduling is that small jobs may have to wait for a longer period of time due to having less priority.
    • Time-Sharing Scheduling: Each program loaded into memory is given a fixed amount of time to execute. Every program that is to be executed has equal processing time so that none of the jobs depends on the time taken by another job while processing.

    Buffering: A buffer is a small, high-speed storage device located in I/O units, used to hold data during input/output operations. It is divided into the input buffer and output buffer. The input buffer accepts data at low speed and releases it at high speed to the CPU, while the output buffer works in reverse order, accepting data at high speed from the main memory and releasing it at low speed to the output device. Buffering is the process of using a buffer during I/O operations. The term buffering is also used while accessing files like audio/video through the internet. Here, it is the process of holding downloaded data before it can be played.

    Virtual Memory: In a multi-programming system, when we use multiple programs at the same time, the space in primary memory (RAM) may not be enough to store all the programs. In this case, the operating system uses free space in secondary memory to store this data. This free space in secondary memory, used to store the contents of primary memory temporarily, is known as virtual memory. An operating system is responsible for creating, managing, and operating virtual memory.

    It is not actual memory but a technique used by the operating system to execute large-sized programs with a small amount of available primary memory. This is done by dividing the larger program into small-sized segments known as pages.

    Long Answer Questions

    1. What is an operating system? Explain its major functions. [HSEB 2061, 2065, 2066, 2070]
    An operating system (OS) is system software that controls and manages computer hardware and software resources, and provides common services for computer programs. Major functions include:

    • Process Management: Handles process creation, scheduling, and termination.
    • Memory Management: Allocates and deallocates memory space as needed by programs.
    • File System Management: Organizes, stores, and retrieves data on storage devices.
    • Device Management: Controls peripheral devices through drivers and manages input/output operations.
    • User Interface: Provides a way for users to interact with the computer, such as GUI or CLI.
    • Security and Access Control: Protects data and restricts unauthorized access.

    2. “An operating system is an interface between human operators and application software”. Justify the statement with examples of operating system known to you. [HSEB 2057]
    The operating system acts as a mediator between the user and the computer hardware. It provides an interface, such as a graphical user interface (GUI) in Windows or macOS, where users can launch and interact with application software without dealing with the complexities of hardware. For example, in Windows, users can click icons to open applications, while in Unix-like systems, a command-line interface (CLI) allows users to type commands to manage files and run programs.

    3. What is an operating system? Explain the different types of operating system with suitable examples. [HSEB 2067]
    An operating system is the software that manages computer resources and provides services to application programs. Different types include:

    • Batch Operating Systems: Process batches of jobs without user interaction (e.g., IBM's early systems).
    • Time-Sharing Systems: Allow multiple users to share system resources simultaneously (e.g., UNIX).
    • Distributed Operating Systems: Manage a group of independent computers and make them appear as a single system (e.g., Amoeba).
    • Real-Time Operating Systems: Provide immediate processing for critical tasks (e.g., VxWorks, QNX).
    • Network Operating Systems: Support networking features (e.g., Novell NetWare, Windows Server).

    An operating system is software that manages computer resources and provides a user interface.

    • GUI (Graphical User Interface):
      • Merits: Easy to use, intuitive, supports multimedia and graphics, good for beginners.
      • Demerits: Consumes more system resources, may be slower for certain operations, less flexible for advanced tasks.
    • CUI (Character User Interface) / CLI (Command Line Interface):
      • Merits: Lightweight, faster execution of commands, more control and scripting capability for advanced users.
      • Demerits: Steep learning curve, less intuitive, error-prone for casual users.

    5. What is an operating system? Describe the types of operating system. [HSEB 2071]
    An operating system is a core software that manages hardware and software resources and provides services for computer programs. The types include:

    • Single-User, Single-Tasking OS: Supports one user at a time performing one task (e.g., early versions of MS-DOS).
    • Single-User, Multi-Tasking OS: Allows one user to run multiple applications simultaneously (e.g., Windows, macOS).
    • Multi-User OS: Supports multiple users at the same time (e.g., UNIX, Linux).
    • Real-Time OS: Provides immediate response for time-critical applications (e.g., embedded systems in medical or industrial fields).
    • Distributed OS: Manages a group of computers and makes them work as a single system (e.g., systems used in networked environments).
